# サイドバー

DayFlow のサイドバー体験を 2 つのデモで紹介します。標準 UI のまま使うパターンと、完全カスタムに置き換えるパターンを比較しながら確認してください。

## デフォルトレイアウト

`useSidebar` を有効にすると、カレンダーごとの可視状態トグルやカラー表示、折りたたみボタン付きのサイドバーが即座に組み込まれます。

<SidebarBasicsShowcase />

## カスタムレイアウト

`render` 関数を渡すと、チーム別フィルターやアナリティクスなど好きな UI に変えられます。それでも内部状態は同じ `calendar` オブジェクトを共有するため、カレンダーの表示/非表示や日付選択と同期したままです。

<SidebarCustomShowcase />

```tsx copy
import type { CalendarSidebarRenderProps } from '@dayflow/core';

const CustomSidebar = ({
  calendars,
  toggleCalendarVisibility,
  toggleAll,
  isCollapsed,
  setCollapsed,
}: CalendarSidebarRenderProps) => {
  if (isCollapsed) {
    return <button onClick={() => setCollapsed(false)}>展開</button>;
  }

  return (
    <aside className="flex h-full flex-col gap-4 p-4">
      <header className="flex items-center justify-between">
        <h3 className="text-sm font-semibold">カレンダー</h3>
        <div className="space-x-2">
          <button onClick={() => toggleAll(true)}>全表示</button>
          <button onClick={() => toggleAll(false)}>全非表示</button>
        </div>
      </header>
      <ul className="space-y-2">
        {calendars.map(calendar => (
          <li key={calendar.id} className="flex items-center gap-2">
            <input
              type="checkbox"
              checked={calendar.isVisible}
              onChange={() =>
                toggleCalendarVisibility(calendar.id, !calendar.isVisible)
              }
            />
            <span>{calendar.name}</span>
          </li>
        ))}
      </ul>
    </aside>
  );
};

const calendar = useCalendarApp({
  useSidebar: {
    enabled: true,
    width: 320,
    render: props => <CustomSidebar {...props} />,
  },
});
```

### 見どころ

- 可視状態を切り替えるとカレンダー側もリアルタイムに反映
- ビューを切り替えてもカスタムサイドバーに表示する「次の予定」や KPI を同期可能
- 集中したいときは折りたたんで表示領域を広げられる

詳細な設定オプションは [導入ガイドのサイドバー章](/docs-ja/introduction/sidebar) も参照してください。
